# LLM-Driven Three-Pass Recommendation Enrichment - Implementation Guide

## What Was Implemented

A **pure LLM-based enrichment system** that transforms shallow recommendations into deeply analytical, production-ready insights. Everything is generated through intelligent LLM prompting—no hardcoded logic, pricing data, or deterministic algorithms.

## Three-Pass Pipeline

```
┌─────────────────────────────────────────────────────────────────┐
│ PASS 1: KB LINKER (Agent 1)                                     │
│ Maps each resource to applicable AWS cost optimization strategies│
│ Input: Service inventory + cost anchors + KB                     │
│ Output: [ {resource, strategy, evidence, reduction%} ]           │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│ PASS 2: GENERATOR (Agent 2)                                     │
│ Creates initial recommendations                                  │
│ Input: Agent 1 mappings + cost anchors + dependency graph       │
│ Output: [ {resource, action, savings, title, finding} ]         │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│ PASS 3: ENRICHMENT (Agent 3) ← NEW                              │
│ LLM-driven deep analysis on each recommendation                 │
│ Input: Initial recommendation + resource context + KB + deps    │
│ Output: Enhanced recommendation with:                            │
│   - detailed_metrics_analysis                                    │
│   - cost_breakdown_analysis (with formulas)                      │
│   - implementation_roadmap (AWS CLI steps)                       │
│   - risk_assessment (blast radius + mitigation)                  │
│   - business_impact_narrative                                    │
│   - kb_mapping (pillars + docs + compliance)                     │
└─────────────────────────────────────────────────────────────────┘
                            ↓
                   Enriched Recommendations
                (100+ cards with full analysis)
```

## Key Design Principles

### 1. Pure LLM-Driven
- **Everything** is generated by the LLM through prompts
- No hardcoded AWS pricing rates or formulas
- No deterministic business logic
- LLM reasons about metrics, costs, and risks contextually

### 2. Backward Compatible
- Enrichment fields are **optional** additions
- If enrichment fails, system falls back to non-enriched recommendations
- Existing API clients continue to work unchanged
- Old recommendations without enrichment still display correctly

### 3. Scalable
- Works for **any AWS service** (EC2, RDS, S3, Lambda, etc.)
- LLM adapts the analysis based on service type
- No need to add new code when AWS services are added
- Enrichment happens for **all recommendations** (100%+), not just top 10

### 4. Contextual & Intelligent
- LLM has full resource context (metrics, dependencies, config)
- Considers blast radius of changes
- Aligns recommendations with AWS Well-Architected Framework
- Generates service-specific AWS CLI commands

## How It Works at Runtime

### Step 1: Initial Recommendation Generation (Pass 1 & 2)
```
Invoke: POST /api/recommendations/generate-bg
  ↓
Pass 1: KB Linker LLM call
  - Input: Service inventory + cost anchors + KB
  - Output: [ {resource → best practice strategy} ]
  ↓
Pass 2: Generator LLM call
  - Input: Agent 1 output + cost anchors + graph
  - Output: [ {resource, action, savings, title, findings} ]
```

### Step 2: Enrichment Pass (Pass 3 - NEW)
```
For each recommendation from Pass 2:
  ↓
1. Extract resource context
   - Find resource in graph
   - Gather metrics, cost, config, attributes
   ↓
2. Extract dependencies
   - Find which services depend on this resource
   - Map dependent services (blast radius)
   ↓
3. Get service KB
   - Load AWS best practices for this service type
   - Get optimization ranges and industry benchmarks
   ↓
4. Build enrichment prompt
   - Section 1: The initial recommendation
   - Section 2: Detailed resource context
   - Section 3: Dependency map
   - Section 4: AWS KB for service
   - Request: Deep analysis (metrics, cost, steps, risks, narrative, KB)
   ↓
5. Call LLM (Ollama/Gemini)
   - LLM analyzes the recommendation with full context
   - LLM reasons about: metrics, costs, risks, implementation
   - LLM generates: JSON with enriched fields
   ↓
6. Parse JSON response
   - Extract enriched fields
   - Merge into recommendation object
   ↓
7. Move to next recommendation
```

### Step 3: Storage and Display
```
Enriched recommendations
  ↓
Normalize (flatten to unified schema)
  ↓
Store in RecSnapshot table + Redis cache
  ↓
Return to API with full enrichment
  ↓
Frontend displays: Metrics → Cost Breakdown → Steps → Risks → Narrative
```

## Configuration & Tuning

### LLM Model Selection
Set in environment variables (used for enrichment):
```bash
# Use local Qwen 2.5 7B (default)
OLLAMA_URL=http://localhost:11434
FINOPS_MODEL=qwen2.5:7b

# Or use Gemini Flash for richer analysis
USE_GEMINI=true
GEMINI_API_KEY=your-api-key
```

### Enrichment Parameters
In `enrich_recommendations_pass_3()`:
- `temperature=0.2` - Lower temp for factual analysis (no hallucination)
- `max_tokens=3000` - Budget for detailed analysis per recommendation

### Graceful Degradation
If enrichment fails for any reason:
- System logs the error
- Recommendation is returned un-enriched
- Pipeline continues to next recommendation
- Users still get original recommendation with basic fields

## Troubleshooting

### Issue: "enriched fields not present"
**Cause**: Enrichment failed silently
**Solution**: Check logs for `[PASS 3]` errors; ensure LLM is configured correctly

### Issue: "AWS CLI commands have wrong syntax"
**Cause**: LLM hallucinated syntax
**Solution**: Lower temperature (was 0.2, try 0.1); update KB with more examples

### Issue: "Enrichment taking too long"
**Cause**: LLM API latency or network issues
**Solution**: Implement timeout (30s); fall back to non-enriched; consider batch processing

### Issue: "Cost calculations don't match CUR"
**Cause**: LLM used outdated pricing
**Solution**: Update context_parts with latest cost anchors from CUR; re-run enrichment
